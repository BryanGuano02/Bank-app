 # Requisitos No Funcionales (RNF)

 ## RNF-01 — Rendimiento

 - **RNF-01.1:** El sistema debe mostrar consultas de saldo en menos de 2 segundos.
 - **RNF-01.2:** El sistema debe registrar un movimiento en menos de 2 segundos.

 ## RNF-02 — Disponibilidad

 - **RNF-02.1:** El sistema debe estar disponible el 99% del tiempo mensual.

 ## RNF-03 — Usabilidad

 - **RNF-03.1:** Las interfaces deben ser intuitivas y accesibles para usuarios no técnicos.
 - **RNF-03.2:** Los errores deben mostrarse en lenguaje claro y no técnico.

 ## RNF-04 — Seguridad

 - *(Sin roles ni auditoría, respetando tu especificación)*
 - **RNF-04.1:** La información personal debe viajar cifrada mediante HTTPS.
 - **RNF-04.2:** Los datos sensibles deben almacenarse en formato seguro.

 ## RNF-05 — Mantenibilidad

 - **RNF-05.1:** El código debe seguir un estándar de estilo definido.
 - **RNF-05.2:** La arquitectura debe permitir extender tipos de cuentas en el futuro.
